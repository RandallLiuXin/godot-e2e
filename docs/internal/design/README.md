# Design Documents

Long-form design notes for non-trivial features, sized between an ADR (a few bullets) and the final user-facing docs.

A design doc lives here while a feature is being shaped: trade-offs considered, alternatives rejected, open questions, file-level change plan. Once the feature ships, the doc stays as the historical record.

When user-facing docs (under `docs/api-reference.md`, `docs/architecture.md`, etc.) are updated for the same feature, link them from the design doc rather than duplicating content.

Current documents will be linked here as they are added.
